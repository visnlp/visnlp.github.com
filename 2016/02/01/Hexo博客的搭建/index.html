Hexo博客的搭建

之前就搭建过一个hexo博客，感觉这种写博客的方式特别有意思，最近在搭建这个博客时，又查了好多资料，遇到各种问题才成功的。因此有必要在博客第一篇就将Hexo博客的搭建过程记录下来。

环境准备

  • node.js
    由于Hexo是一个基于Node.js的静态博客程序，因此node.js是必不可少的。安装node.js只需要去官网下载即可。
  • Git
    Hexo编译上百篇文字只需要几秒，生成的静态网页可以直接放到GitHub，BAE，SAE等各种平台上。目前博客选择部署到github上，因此需要安装Git。
  • MarkdownPad
    Hexo写博客使用Markdown语法，感觉MarkdownPad比较舒服，界面分左右两栏，效果实时可见。

Github部署准备步骤

  1. 注册Github账号。
  2. Github可以用来管理自己的代码，每个项目对应一个仓库。我们需要给自己的博客建一个仓库，仓库名必须是”your_username.github.com”。

  3. 设置自己的用户名和密码。在文件夹右键选择“Git Bash”，输入一下两条命令：

    git config –global user.email “your_email@xxx.com”
    git config –global user.name “your_username”

  4. 添加SSH_Key。Git是分布式的代码管理工具，远程的代码管理是基于SSH的，所以要使用远程的Git则需要SSH的配置。生成密钥：

    ssh-keygen -t rsa -C “your_email@xxx.com”

    之后遇到选项全部Enter跳过即可。

  5. 在上步执行成功后，会在当前目前下生成两个文件id_rsa和id_rsa.pub。然后在github上的Add SSH Key上的Key文本框粘贴id_rsa.pub里的内容，Title可随意。

  6. 可以使用下面的命令验证一下：

    ssh -T git@github.com

Hexo安装和使用

在准备了这么久之后，终于到了正式开始Hexo的时候了，哈哈！

  • 安装hexo
    npm install -g hexo
  • 初始化，先cd到博客的搭建目录，然后执行：
    hexo init
  • 在上一步执行后会显示：
    INFO Copying data to D:\hexo
    INFO You are almost done! Don’t forget to run ‘npm install’ before you start blogging with Hexo!
    因此还要执行以下命令来安装依赖包：
    npm install
  • 下面介绍hexo常用命令：
    • hexo news [layout] “name” //新建文章
    • hexo server //启用本地服务，可进行文章预览调试。
    • hexo g //生成博客的静态文件
    • hexo d //部署，部署到哪可以设置
